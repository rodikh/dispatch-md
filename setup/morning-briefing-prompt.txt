## STEP 1 — Get dates and lookback window

Run in bash:
  TODAY=$(date '+%Y-%m-%d')
  TODAY_DISPLAY=$(date '+%A, %B %-d')
  YESTERDAY=$(date -v-1d '+%Y-%m-%d')
  TOMORROW=$(date -v+1d '+%Y-%m-%d')
  DOW=$(date '+%u')   # 1=Mon 2=Tue 3=Wed 4=Thu 5=Fri 6=Sat 7=Sun

  # Adjust the lookback so Monday (or your first workday) covers the whole weekend.
  # Default: on the first workday of the week, look back to Friday midnight; otherwise 24h.
  if [ "$DOW" = "1" ]; then
    SINCE=$(date -v-3d -v0H -v0M -v0S -u '+%Y-%m-%dT%H:%M:%SZ')
    WINDOW_LABEL="the weekend (Fri–Sun)"
  else
    SINCE=$(date -v-1d -u '+%Y-%m-%dT%H:%M:%SZ')
    WINDOW_LABEL="the last 24 hours"
  fi

  # GitHub PR comment window (slightly wider for safety)
  if [ "$DOW" = "1" ]; then
    GH_SINCE=$(date -v-4d -u '+%Y-%m-%dT%H:%M:%SZ')
  else
    GH_SINCE=$(date -v-2d -u '+%Y-%m-%dT%H:%M:%SZ')
  fi

## FORMATTING RULE — PR and ticket references

When a PR number or ticket key appears **without surrounding context that already makes it
clear what it is** — e.g. in priority bullets, standup paragraphs, blockers, or free-form
prose — append a short 2–3 word descriptor in parentheses right after it.

Examples:
  - "Follow up on PR #1234 (fix panel crash)"
  - "PROJ-5678 (add color tokens) is blocked on design sign-off"

You do NOT need the descriptor inside dedicated tables (PR Status, tracker) where the title
column already provides the context.

## STEP 3 — Gather live data

### GitHub (if `gh` CLI is configured)

```bash
# Open PRs with age and review state
gh pr list --repo {{GITHUB_REPO}} --author "@me" --state open \
  --json number,title,headRefName,reviewDecision,url,createdAt,isDraft \
  --jq '.[] | {number, title, branch: .headRefName, review: (.reviewDecision // "PENDING"),
    draft: .isDraft, url, age_days: ((now - (.createdAt | fromdateiso8601)) / 86400 | floor)}'

# New comments from others — use GH_SINCE computed above
gh api "repos/{{GITHUB_REPO}}/issues/comments?since=${GH_SINCE}&per_page=50" \
  --jq '[.[] | select(.user.login != "{{GITHUB_USER}}") | {author:.user.login, url:.html_url, snippet:.body[:200]}]'
```
If GitHub isn't configured, skip this section.

### Issue tracker (e.g. Jira) — try MCP

If a Jira/issue-tracker MCP is available, run two queries:

**Query 1 — Active tickets:**
  JQL: `assignee = "{{USER_EMAIL}}" AND statusCategory != Done ORDER BY updated DESC`

**Query 2 — Recently completed:**
  JQL: `assignee = "{{USER_EMAIL}}" AND statusCategory = Done AND updated >= -2d ORDER BY updated DESC`

vault project file). Group by project/theme. If unavailable: note "Tracker requires Claudian session."
